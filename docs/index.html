RS3 GIM Companion

A private companion web app for RuneScape 3 Group Ironman teams. Track hiscores, XP gains, drops, gear loadouts, and achievements — all in one place.

v1.2.0 RS3 · Not OSRS Discord Support
💡

RS3 GIM Companion is live at groupiron.com and can also be self-hosted on any Node.js server. All data stays on your own infrastructure.

Quick Start

1. Find Your Group

Open the app and click 👥 Find or Manage Groups in the sidebar. Search for your GIM group name — the app looks it up against the RS3 hiscores API and auto-populates all members.

2. Claim Your Group

A newly added group is unclaimed (read-only for everyone). Click 🔒 Claim group in the header. A secret code is generated once — copy it and share it with your team on Discord. Any browser that enters this secret can edit; everyone else gets read-only access.

✅

You can't recover a lost secret. If you lose it, the group must be unclaimed and re-claimed. Keep it somewhere safe (e.g. a private Discord channel).

3. Set Your Character Name

After unlocking, click the 👤 Set your name button in the header and pick your RSN. This tells the app which gear loadout belongs to you — you can only edit your own character's equipment.

4. Sync Data

Click ↻ Sync All to pull the latest hiscores for all members. The activity feed (RuneMetrics) is updated automatically every 2 hours in the background — it is intentionally kept separate from Sync All to avoid rate-limiting the Jagex API. XP snapshots are taken daily at midnight.

Access Control

StateWho can viewWho can edit
UnclaimedEveryoneEveryone (no password set)
Claimed, lockedEveryoneNobody (until unlocked)

Gear loadouts have an additional permission layer: even when a group is unlocked, you can only edit your own character's gear. Other players' slots are view-only.

📊 Overview Tab

The landing page for your group. Defaults to this tab on load.

Member Cards

One card per player showing RS3 avatar, combat level, total level, and total XP. The YOU badge highlights your character. Click any card to focus — the Goals and Activity panels filter to that player.

Group Stats

  • XP bars — progress toward group XP milestones
  • Skills matrix — all 29 RS3 skills across all players; highest level per skill is gold-highlighted; includes Necromancy and Summoning
  • Combat tab — shows Combat level with RS3 skill icons

Goals & Activity

Side-by-side panels (or stacked on mobile). Goals support four types: Level (with XP-to-go tracking), Item, Quest, and Custom. The activity feed shows the RuneMetrics history for the selected player — entries are stored permanently in the database and auto-refreshed every 2 hours.

🏆 Group Vault

The vault shows every item the group has logged — from the drop log and the automatic RuneMetrics activity feed. It also tracks confirmed worn gear so the group always knows who has what.

Drop Cards

Each unique item gets a card showing who dropped it, the boss source, value, and date. Cards are sorted by most recent by default. Additional sort/filter options: Dupes · Value · A–Z · filter by player or boss.

  • WORN — someone in the group has confirmed they wear this item
  • DUPE — two or more players have obtained this item

When a drop card matches a confirmed gear entry, it shows a green row: "✅ Worn by PlayerName · slot · claimed date"

Confirmed Gear section

Below the drop list, every confirmed gear entry is listed — even items never logged as drops. This is how gear set in the Gear Loadouts panel immediately shows in the Vault without needing a corresponding drop entry.

⚡

Confirming an item in Gear Loadouts updates the Vault in real time — no refresh needed.

Group Notes

Click 📝 Group Notes (top right of the vault) to open a slide-in overlay — a shared pinboard for strategies, loot distribution rules, or session plans. Auto-saves with a 1.5 s debounce. Close with Escape or clicking the backdrop.

⚔️ Gear Loadouts

Sits side-by-side with the Group Vault in the same tab. Tracks each player's equipment across five combat styles.

Combat Styles

Melee · Ranged · Magic · Necromancy · Hybrid — each has its own 12-slot grid (head, cape, neck, ammo, weapon, body, offhand, legs, gloves, boots, ring, pocket).

Slot States

Border colourDotMeaning
Green✅Confirmed — player owns / wears this item

Item Requirements

Every suggested item has wiki-verified equip requirements. Items you can't use show a 🔒 lock with the missing skill levels (e.g. Defence 80/90). Missing requirements can be added as Goals directly from the picker with one click.

Quest-locked items cascade correctly — e.g. Salve amulet (e) requires Lair of Tarn Razorlor, which itself requires completing Haunted Mine (Agility 15, Crafting 35) and having Slayer 40.

Permissions

Only your own character is editable (the one you set with 👤 Set your name). When viewing a teammate's gear the panel shows: "🔒 View only — switch to your character to edit gear".

Quick-fill & Clear

  • ⚡ Quick-fill — fills all empty slots with the best item you can currently equip, marked as Planning
  • 🗑 Clear gear — removes all items for the current style in one click

💎 Items & Drops

Single page showing three panels simultaneously — no sub-tabs to switch between.

Item Requests (left panel)

Your group's wish-list. Each request has: item name, boss/activity, requesting player, priority (🔴 High / 🟡 Medium / 🟢 Low), quantity, and optional notes.

Boss names show a colour-coded readiness chip: 🟢 Can kill now / 🟠 Close / 🔴 Missing requirements. Hover the chip to see a detailed popup with exact skill requirements and which players are eligible.

Mark requests as ✓ Obtained to archive them; filter between Pending / Obtained / All.

Drop Log (middle panel)

Manual drop logging plus auto-detected entries from the RuneMetrics activity feed (e.g. "I found a Dragon chainbody" lines are parsed on every sync). Filter by player.

Boss Overview (right sidebar, 20%)

A compact sidebar showing every boss that has pending requests. For each boss:

  • Colour-coded readiness chip
  • Items being requested (tags)
  • Who needs them
  • Who can attempt the boss (or how many levels away the closest player is)

📋 Achievement Diaries

Tracks diary completion across all 13 RS3 regions × 4 tiers (Easy / Medium / Hard / Elite).

Grid View (default)

Regions as rows, tiers as columns. Each cell shows coloured player dots — one per player, filled if they've completed that diary tier.

Player View

Per-player breakdown with tier buttons and completion dates. Completion percentage bar per player.

Auto-detection

Diary completions are detected from the RuneMetrics activity feed on every sync (e.g. "I completed the Easy Varrock diary"). Any cell can also be toggled manually if the feed entry was missed.

🏅 Leaderboards

Skill Mastery

🔔 Discord Notifications

Connect your group to a Discord channel to receive automatic notifications when things happen in-game. Each group stores its own webhook and event preferences.

Setup

  1. Unlock your group (enter the group secret)
  2. Click the 🔔 button in the header
  3. In Discord: right-click a channel → Edit Channel → Integrations → Webhooks → New Webhook → copy the URL
  4. Paste the URL, choose which events to enable, click ✓ Save
  5. Click 📨 Test to confirm the connection — a test embed will appear in your Discord channel immediately

Notification Events

EventDefaultTrigger
🎉 Level 99s & 120sOnDetected from RuneMetrics activity feed on every sync
📋 Achievement DiariesOnWhen a diary tier completion is detected or manually toggled
⚔️ Boss First KillsOnFirst time a player's kill count for a boss goes above zero
🎯 Group GoalsOnWhen any goal is marked as complete
🎁 DropsOffEvery drop auto-detected from activity feed — can be noisy
🔒

The webhook URL is stored server-side and only readable by browsers with the group secret. Only group owners can view or change it.

Local Development

git clone https://github.com/thosjo95/RS3-GIM-Companion.git
cd RS3-GIM-Companion
npm run install:all   # installs root + server + client deps
npm run dev           # API on :3001, Vite dev server on :5173

Open http://localhost:5173 — /api requests are proxied to the backend automatically by Vite.

Production Deploy

ssh root@YOUR_SERVER_IP
bash /root/deploy.sh
# Script: git pull → npm run build → pm2 restart app
⚡

Backend-only change? Skip the Vite rebuild: git pull && pm2 restart rs3-gim
Use deploy.sh whenever any file under client/ changes. When in doubt, run the full script — it's always safe.

🛠️ Admin FAQ

Common issues encountered when running the server or managing groups.

Group shows 0 members / members not loading

Symptom: A group was added and claimed, but the member list is empty — even though another unclaimed group with the same name has all the players.

Cause: Players were added to the group before claiming. Claiming created a new database entry with a different id, leaving players attached to the old unclaimed record.

Step 1 — diagnose: Find both groups and note their IDs.

ssh root@YOUR_SERVER_IP
node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const groups = db.prepare(\`
  SELECT id, name, password_hash,
    (SELECT COUNT(*) FROM players WHERE group_id = g.id) AS player_count
  FROM groups g WHERE name LIKE '%YOUR_GROUP_NAME%'
\`).all();
console.log(JSON.stringify(groups, null, 2));
"

You'll see two rows: one with player_count > 0 and no password_hash (OLD), and one with a hash but 0 players (NEW/claimed). Note both id values.

Step 2 — fix: Move players to the claimed group and delete the orphan.

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const moved = db.prepare('UPDATE players SET group_id = NEW_ID WHERE group_id = OLD_ID').run();
const del   = db.prepare('DELETE FROM groups WHERE id = OLD_ID').run();
console.log('Moved:', moved.changes, 'players. Deleted:', del.changes, 'group(s).');
"

Replace OLD_ID and NEW_ID with the values you found. Refresh the app — all members appear immediately.

Activity feed is empty

Cause: The activity feed is populated by a 2-hour background cron (RuneMetrics API). ↻ Sync All only updates hiscores — it does not touch RuneMetrics. If the cron hasn't run yet since the server started, the feed will be empty.

Check the last cron run:

pm2 logs rs3-gim --lines 50

To trigger immediately without waiting (no UI button — server-side only):

# Replace 5 with your group's numeric ID
curl -s -X POST http://localhost:3001/api/players/sync-activities/5 | python3 -m json.tool

This fetches RuneMetrics for every player with a 1-second delay between requests and writes new activities to the database straight away.

RuneMetrics returns 502 / 503 errors

Cause: Jagex's RuneMetrics endpoint is temporarily down or overloaded — this is a server-side problem at Jagex, not a rate-limit issue and not a bug in the app.

Solution: Wait and retry. The 2-hour cron will pick it up automatically. You can also re-run the manual curl once Jagex recovers. No code changes are needed.

ℹ️

502 = Jagex server error (just wait). 429 = rate limited (reduce sync frequency). In practice, at 1 req/sec with a 2-hour window the app is well within safe limits for thousands of players.

A group lost their secret / need to re-claim

Symptom: The group owner lost their secret code and can no longer edit the group.

Fix: Clear the password_hash — this resets the group to unclaimed so they can claim it again and get a new secret. All data (players, goals, drops, gear) is untouched.

Step 1 — list all groups to find their ID:

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const groups = db.prepare('SELECT id, name, password_hash, (SELECT COUNT(*) FROM players WHERE group_id = g.id) AS players FROM groups g ORDER BY id').all();
console.log(JSON.stringify(groups, null, 2));
"

Step 2 — clear the password (replace 26 with their actual group ID):

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const result = db.prepare('UPDATE groups SET password_hash = NULL WHERE id = ?').run(26);
console.log('Updated rows:', result.changes);
"

If it prints Updated rows: 1 you're done. Tell them to refresh — 🔒 Claim group reappears in the header. They click it, get a new secret shown once, and save it somewhere safe (e.g. a private Discord channel).

Permanently delete a group

Removes the group and all its players from the database (cascade delete).

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const result = db.prepare('DELETE FROM groups WHERE id = ?').run(GROUP_ID);
console.log('Deleted groups:', result.changes);
"

Replace GROUP_ID with the target ID found from the listing above.
