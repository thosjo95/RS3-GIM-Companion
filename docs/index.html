RS3 GIM Companion

A private companion web app for RuneScape 3 Group Ironman teams. Track hiscores, XP gains, goals, gear loadouts, vault items, and achievements — all in one place.

v1.9.2 RS3 · Not OSRS Discord Support
💡

RS3 GIM Companion is live at groupiron.com and can also be self-hosted on any Node.js server. All data stays on your own infrastructure. New users are guided through features with interactive per-tab product tours.

Quick Start

1. Find Your Group

Open the app and click Find or Manage Groups in the sidebar. Search for your GIM group name — the app looks it up against the RS3 hiscores API and auto-populates all members.

2. Claim Your Group

A newly added group is unclaimed (read-only for everyone). Click Claim group in the header. A secret code is generated once — copy it and share it with your team on Discord. Any browser that enters this secret can edit; everyone else gets read-only access.

✅

You can't recover a lost secret. If you lose it, the group must be unclaimed and re-claimed. Keep it somewhere safe (e.g. a private Discord channel).

3. Set Your Character Name

After unlocking, click Set your name in the header and pick your RSN. This tells the app which gear loadout belongs to you — you can only edit your own character's equipment.

4. Sync Data

Click Sync All to pull the latest hiscores for all members. The activity feed (RuneMetrics) is updated automatically every 2 hours in the background. XP snapshots are taken daily at midnight.

Group Management

Click Find or Manage Groups in the sidebar to open the group manager. Each group row displays the correct RS Wiki GIM badge icon (Regular, Competitive, or Unranked), and the active group's badge also appears in the sidebar and the page subheader.

Look Up Group

Enter the exact group name and select the type (Regular / Competitive / Unranked) and member count — the app validates against the RS3 GIM hiscores before importing. The name input placeholder shows a random name from groups already tracked in the app for reference.

Browse Groups Already Tracked

Expand the Browse groups already tracked panel to scroll or search groups already in the app — sorted by most recently active. Click any row to open it in read-only mode without saving it to your sidebar.

Custom Groups Dev Preview

A Custom Group is a non-GIM group for regular RS3 players who want a shared dashboard without a formal Group Ironman setup. Select Custom Group in the type picker when creating a new group. Members are added manually by RSN, and all features work identically — hiscores sync, XP gains, goals, vault, Discord notifications, achievement diaries.

  • Shown with a teal C badge throughout the UI instead of a GIM badge image
  • The app auto-suggests a unique RS-themed name (e.g. Shadow Vanguard, Gilded Champions) that is guaranteed not to conflict with any existing group — editable before confirming
  • A dismissable info banner appears on the dashboard for custom groups
  • Activity feed and drops require a public RuneMetrics profile (same as GIM groups)

Starring / Favouriting Groups

Groups are only pinned to your sidebar when you explicitly save them — browsing a group does not auto-pin it. Three ways to save a group:

  • ☆ Star button on any browse result row
  • ⭐ Save to your groups button in the guest banner (shown when viewing a non-saved group)
  • Automatically when you claim or unlock a group

A saved group appears in your sidebar permanently across sessions. Unsaved / guest groups are viewable but disappear from the sidebar on refresh.

Goal Browser

Goals are displayed in a filterable list. Filter controls along the top let you narrow by:

  • Status — Active, Complete, Vaulted, or All
  • Type — any of the six goal types above
  • Priority — High, Medium, or Low
  • Player — per-member filter chips (same chip design used across the entire app)

Active Goals Board

The Active Goals section shows all in-progress goals as cards organised into three columns: Not Started, In Progress, and Blocked. Each card has a set of action buttons:

  • ✏️ Edit — opens a pre-filled goal modal so you can update the title, description, priority, or any other field without losing context
  • 🚫 Blocked — moves the card to the Blocked column (hidden once already blocked or complete)
  • ▶ Advance — moves the card to the next status (Not Started → In Progress → Complete)
  • ✕ Delete — removes the goal entirely (with confirmation)

Cards can also be drag-and-dropped directly between columns — grab any card and drop it into a different column to update its status instantly. The target column highlights with a gold border while you drag. Empty columns show a "Drop here" hint during a drag.

Group Vault

The vault is the left panel of the Vault tab. It shows every item the group has logged — from the activity feed, manual adds, and completed goals — displayed as a compact tile grid.

Discord Notifications

Self-Hosting

Requirements

  • Node.js 22+
  • Any Linux VPS (Ubuntu 22.04 recommended)
  • Nginx for reverse proxy + SSL
  • PM2 for process management

Local Development

git clone https://github.com/thosjo95/RS3-GIM-Companion.git
cd RS3-GIM-Companion
npm run install:all   # installs root + server + client deps
npm run dev           # API on :3001, Vite dev server on :5173

Open http://localhost:5173 — /api requests are proxied to the backend automatically by Vite.

Seed Gear & Item Data

After first deploy (or after pulling to a new server), seed the RS3 gear and item database so gear slot icons, vault icons, and all item suggestions display correctly:

cd /var/www/RS3-GIM-Companion
node server/scripts/seedRs3Data.js

This seeds 410+ gear items (with requirements and icons) plus 168 skill milestones. It uses INSERT OR REPLACE and is safe to re-run at any time — no restart needed.

Production Deploy

ssh root@YOUR_SERVER_IP
bash /root/deploy.sh
# Script: git pull → npm run build → pm2 restart app

# After deploy, re-seed if gear data changed:
cd /var/www/RS3-GIM-Companion && node server/scripts/seedRs3Data.js
⚡

Backend-only change? Skip the Vite rebuild: git pull && pm2 restart rs3-gim
Use deploy.sh whenever any file under client/ changes. When in doubt, run the full script — it's always safe.

Admin Portal

The admin portal is a separate, password-protected management interface accessible at /admin on any RS3 GIM Companion deployment. It gives server operators a full UI for managing groups, reviewing data submissions, inspecting reference tables, and running maintenance operations — without needing SSH or direct database access.

🔒

Access: Navigate to https://groupiron.com/admin (or http://localhost:5173/admin in dev). Log in with your admin credentials. Sessions last 8 hours and are stored in sessionStorage — they survive page refreshes but not closing the browser tab.

Authentication

Admin credentials are stored in the admin_users database table. Passwords are hashed with PBKDF2-SHA512 (600,000 iterations) with a random salt. Tokens are signed with HS256 (HMAC-SHA256) and expire after 8 hours. The JWT secret is generated on first run and persisted to server/data/jwt_secret.bin so tokens survive server restarts.

Failed login attempts are rate-limited: 5 failed attempts per IP triggers a 15-minute lockout. Successful login clears the counter.

📥 Queue — Data submission review

The Queue tab is the default view. It lists all pending data submissions — proposed changes to RS3 reference tables (quests, bosses, gear items, skill milestones, slayer creatures, gear paths) that have been queued but not yet applied to the database.

  • Filter by status: Pending · Approved · Rejected · Superseded
  • Each row shows the action (CREATE / UPDATE / DELETE), the target table, the record ID, who submitted it, and when
  • Click Review → to open the review modal: a side-by-side JSON diff (current record vs proposed), an optional review note field, and ✓ Approve & Apply / ✗ Reject buttons
  • Approving writes the change to the database in a transaction; if the same record has other pending submissions they are automatically marked Superseded
  • The header shows a live count badge — 📥 Queue (3) — whenever there are unreviewed submissions

👥 Groups — Group overview

The Groups tab shows every group registered on the platform in a searchable, sortable table.

  • Stat cards at the top: Total groups · Claimed · Unclaimed · Total players · Avg players/group · ⚠ Sync errors · 🔔 Discord wired
  • Table columns: ID · Name · Group RSN · Type · Size · Players · Claimed ✓ · Errors · Discord · Last Activity · Created
  • Search by group name, ID, or group RSN
  • Click any row to expand it and see all players in that group — player IDs, RSNs, combat levels, last synced time, and any active sync error

Admin FAQ

Common issues encountered when running the server or managing groups. Most of these can now be resolved directly in the Admin Portal without SSH access.

Group shows 0 members / members not loading

⚙️

Easier via Admin Portal: Use Maintenance → Browse Groups to find both group IDs, then Move Players Between Groups to move them, and Delete Empty Group to remove the orphan — no SSH required.

Symptom: A group was added and claimed, but the member list is empty — even though another unclaimed group with the same name has all the players.

Cause: Players were added to the group before claiming. Claiming created a new database entry with a different id, leaving players attached to the old unclaimed record.

Step 1 — diagnose: Find both groups and note their IDs.

ssh root@YOUR_SERVER_IP
node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const groups = db.prepare(\`
  SELECT id, name, password_hash,
    (SELECT COUNT(*) FROM players WHERE group_id = g.id) AS player_count
  FROM groups g WHERE name LIKE '%YOUR_GROUP_NAME%'
\`).all();
console.log(JSON.stringify(groups, null, 2));
"

You'll see two rows: one with player_count > 0 and no password_hash (OLD), and one with a hash but 0 players (NEW/claimed). Note both id values.

Step 2 — fix: Move players to the claimed group and delete the orphan.

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const moved = db.prepare('UPDATE players SET group_id = NEW_ID WHERE group_id = OLD_ID').run();
const del   = db.prepare('DELETE FROM groups WHERE id = OLD_ID').run();
console.log('Moved:', moved.changes, 'players. Deleted:', del.changes, 'group(s).');
"

Replace OLD_ID and NEW_ID with the values you found. Refresh the app — all members appear immediately.

Activity feed is empty

⚙️

Easier via Admin Portal: Use Maintenance → Trigger Activity Sync — enter the group ID and click Run. Shows per-player results with new activity counts instantly.

Cause: The activity feed is populated by a 2-hour background cron (RuneMetrics API). Sync All only updates hiscores — it does not touch RuneMetrics.

Check the last cron run:

pm2 logs rs3-gim --lines 50

To trigger immediately without the portal (server-side only):

# Replace 5 with your group's numeric ID
curl -s -X POST http://localhost:3001/api/players/sync-activities/5 | python3 -m json.tool

RuneMetrics returns 502 / 503 errors

Cause: Jagex's RuneMetrics endpoint is temporarily down, or the player's profile is set to Private.

Fix: Ensure each player has their RuneMetrics profile set to Public in-game: Settings → Gameplay → Chat & Social → RuneMetrics Event Log. For server errors, wait and the 2-hour cron will retry automatically.

ℹ️

502 = Jagex server error (just wait). 429 = rate limited (reduce sync frequency). At 1 req/sec with a 2-hour window the app is well within safe limits.

Gear icons missing in the vault or loadouts

⚙️

Easier via Admin Portal: Use Maintenance → Re-run Seed Data — no fields to fill, just click Run. The script output is shown in a collapsible panel so you can confirm it completed.

Cause: The rs3_items table has not been seeded on this server.

Fix (SSH alternative):

cd /var/www/RS3-GIM-Companion
node server/scripts/seedRs3Data.js

The seed script uses INSERT OR REPLACE and is safe to re-run at any time. No restart needed — icons and gear suggestions appear on the next page load.

A group lost their secret / need to re-claim

⚙️

Easier via Admin Portal: Use Maintenance → Reset Group Secret — enter the group ID (find it via Browse Groups) and click Run. Done in one step.

Fix (SSH alternative): Clear the password_hash — this resets the group to unclaimed so they can claim again. All data is untouched.

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
// List groups to find the ID
const groups = db.prepare('SELECT id, name, (SELECT COUNT(*) FROM players WHERE group_id = g.id) AS players FROM groups g ORDER BY id').all();
console.log(JSON.stringify(groups, null, 2));
"

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const result = db.prepare('UPDATE groups SET password_hash = NULL WHERE id = ?').run(GROUP_ID);
console.log('Updated rows:', result.changes);
"

Permanently delete a group

⚙️

Easier via Admin Portal: Use Maintenance → Permanently Delete Group — requires ticking a confirmation checkbox before the Run button activates. All cascades happen automatically.

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const id = GROUP_ID;
db.prepare('DELETE FROM players WHERE group_id = ?').run(id);
const result = db.prepare('DELETE FROM groups WHERE id = ?').run(id);
console.log('Deleted groups:', result.changes);
"

Player shows "Not synced" / hiscores lookup fails

⚙️

Easier via Admin Portal: Use Maintenance → Browse Players in Group to find the player ID, then Fix Player RSN to overwrite the RSN and clear the sync error in one operation.

Cause: The RSN was entered with a non-breaking space instead of a regular space — common when copy-pasting from Discord or mobile. Fix by overwriting the RSN with a clean version:

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
db.prepare('UPDATE players SET rsn = ? WHERE id = ?').run('Correct Name Here', PLAYER_ID);
console.log('Done');
"
curl -s -X POST http://localhost:3001/api/players/PLAYER_ID/sync | python3 -m json.tool
