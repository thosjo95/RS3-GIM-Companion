RS3 GIM Companion

A private companion web app for RuneScape 3 Group Ironman teams. Track hiscores, XP gains, goals, gear loadouts, vault items, and achievements — all in one place.

v1.6.0 RS3 · Not OSRS Discord Support
💡

RS3 GIM Companion is live at groupiron.com and can also be self-hosted on any Node.js server. All data stays on your own infrastructure.

Quick Start

1. Find Your Group

Open the app and click Find or Manage Groups in the sidebar. Search for your GIM group name — the app looks it up against the RS3 hiscores API and auto-populates all members.

2. Claim Your Group

A newly added group is unclaimed (read-only for everyone). Click Claim group in the header. A secret code is generated once — copy it and share it with your team on Discord. Any browser that enters this secret can edit; everyone else gets read-only access.

✅

You can't recover a lost secret. If you lose it, the group must be unclaimed and re-claimed. Keep it somewhere safe (e.g. a private Discord channel).

3. Set Your Character Name

After unlocking, click Set your name in the header and pick your RSN. This tells the app which gear loadout belongs to you — you can only edit your own character's equipment.

4. Sync Data

Click Sync All to pull the latest hiscores for all members. The activity feed (RuneMetrics) is updated automatically every 2 hours in the background. XP snapshots are taken daily at midnight.

Access Control

StateWho can viewWho can edit
UnclaimedEveryoneEveryone (no password set)
Claimed, lockedEveryoneNobody (until unlocked)
Claimed, unlockedEveryoneAny browser with the group secret

Gear loadouts have an additional permission layer: even when a group is unlocked, you can only edit your own character's gear. When viewing another group without being unlocked, all editing UI (slot pickers, add item, delete item, add goal) is hidden — the app is fully read-only for visitors.

Overview Tab

The landing page for your group. Defaults to this tab on load.

Member Cards

One card per player showing RS3 avatar, combat level, total level, and total XP. The YOU badge highlights your character. Click any card to focus — the Goals and Activity panels filter to that player.

Group Stats

  • XP bars — progress toward group XP milestones
  • Skills matrix — all 29 RS3 skills across all players; highest level per skill is gold-highlighted; includes Necromancy and Summoning
  • Combat tab — shows Combat level with RS3 skill icons

Activity Feed

Side panel showing the RuneMetrics history for the selected player — entries are stored permanently in the database and auto-refreshed every 2 hours.

Goals Tab

The Goals tab is the central planning board for your GIM group. It replaces the old Item Requests panel with a more flexible system covering all goal types.

Goal Types

TypeUse case
QuestTrack a quest with all its skill requirements and prereqs
Skill UnlockTarget a skill level for a specific player (with XP-to-go tracking)
Key ItemBoss drops, rare rewards, or key gear pieces to obtain
Item RequestDungeoneering tokens, Archaeology materials, or special currency rewards
Achievement DiaryDiary completion or task set milestone goals
Boss KillCombat milestones or boss completion goals

Goal Browser

Goals are displayed in a filterable list. Filter controls along the top let you narrow by:

  • Status — Active, Complete, Vaulted, or All
  • Type — any of the six goal types above
  • Priority — High, Medium, or Low
  • Player — per-member filter chips (same chip design used across the entire app)

Adding Goals

Click + Add Goal (visible only when the group is unlocked) to open the goal modal. Fill in the type, title, priority, and optional notes. Skill Unlock goals automatically calculate XP remaining. Quest goals let you track individual step completion.

Goal Status

StatusMeaning
ActiveIn progress — shown on the Goals tab and Overview panel
CompleteDone — can be moved to Vaulted to archive
VaultedArchived in the Group Vault achievements section

Goal Suggestions

The app ships with a library of pre-built goal suggestions covering common RS3 GIM priorities — quests, skill milestones, boss targets, and key items. These populate the "Add Goal" picker with one click.

Group Vault

The vault is the left panel of the Vault tab. It shows every item the group has logged — from the activity feed, manual adds, and completed goals — displayed as a compact tile grid.

Item Tiles

Each unique item gets an 80 px tile showing its RS3 wiki icon, name, and the player(s) who own it. Click a tile to expand it for full details; click again to collapse.

  • Dupe badge (gold circle, top-left) — two or more players have logged this item
  • Worn dot (green circle, top-right) — a player has this item confirmed in their gear loadout

Expanded Detail

Clicking a tile opens an inline panel below the tile row showing each drop entry: player name, acquisition date, source boss/activity, quantity, and value. When canWrite is active, a small × button appears on the right of each entry to delete it from the vault. Deletion requires a confirmation prompt.

Filters & Sort

  • Sort buttons — Recent · Dupes · Value · A–Z
  • Search bar — live filter by item name
  • Player chips — one chip per member; click to filter to that player's items (same chip design as Goals and Gear Loadouts)
  • Boss dropdown — filter by source boss (appears only when boss-sourced items exist)

Adding Items Manually

Click + Add Item (visible when unlocked) to open the Add Item modal. Type any item name to search the RS3 wiki — results show the item icon and name. Select the item, choose which player obtained it, and optionally enter a source (e.g. "Nex", "Crafting"). Manual entries are tagged internally and their note is not shown in the vault display.

⚡

Items are also added automatically from the RuneMetrics activity feed on every 2-hour sync. Manual add is for items that weren't detected automatically.

Vaulted Achievements

Below the item grid, any goals with status "Vaulted" appear as achievement cards showing the goal title, owner, and completion date.

Group Notes

Click Group Notes (top right of the vault) to open a slide-in overlay — a shared pinboard for strategies, loot distribution rules, or session plans. Auto-saves with a 1.5 s debounce. Close with Escape or clicking the backdrop.

Gear Loadouts

Sits side-by-side with the Group Vault in the same tab. Tracks each player's equipment across five combat styles.

Player Selector

A row of player chips at the top lets you switch between group members. The active chip is gold-highlighted. Your own character is marked with a small "you" label. This uses the same universal chip design as the vault and goals filters.

Combat Styles

Melee · Ranged · Magic · Necromancy · Hybrid — each has its own 12-slot grid (head, cape, neck, ammo, weapon, body, offhand, legs, gloves, boots, ring, pocket).

Slot States

Border colourMeaning
GreenConfirmed — player owns / wears this item
Gold / amberPlanning — targeted, not yet obtained
DefaultEmpty slot

Item Database (410+ items)

Gear suggestions are pulled live from the database on every page load — no static lists. All four PvM wiki style pages (Melee / Ranged / Magic / Necromancy) are seeded across every equipment slot and tier. Items stored with style: all (pocket items, universal ammo) appear automatically in every combat style's picker.

Cross-Style Item Picker

Each slot picker has two tabs: the default style-specific suggestions, and a "🔍 All items" tab that lists items from other combat styles for that slot. This lets you equip off-style gear (e.g. Pernix's quiver on a Melee setup for prayer bonus) without the recommendations panel ever suggesting cross-style gear.

Item Requirements

Every suggested item has wiki-verified equip requirements. Items you can't use show a lock with the missing skill levels (e.g. Defence 80/90). Missing requirements can be added as Goals directly from the picker with one click.

Permissions

Only your own character is editable (the one set with Set your name). When viewing a teammate's gear the slot picker is hidden. When visiting another group without being unlocked, the entire editing UI is hidden — no slot picker, no quick-fill, no clear button.

Quick-fill & Clear

  • Quick-fill — fills all empty slots with the best item you can currently equip, marked as Planning
  • Clear gear — removes all items for the current style in one click

Achievements Panel

A panel showing the group's overall completion progress across tracked goal categories. Displayed with the same dark panel background as the Goals and Vault tabs.

Progress Summary

Completion bars and counts broken down by goal type — showing how many goals are active, complete, or vaulted across the whole group.

Achievement Diaries

Tracks diary completion across all 13 RS3 regions × 4 tiers (Easy / Medium / Hard / Elite).

Grid View (default)

Regions as rows, tiers as columns. Each cell shows coloured player dots — one per player, filled if they've completed that diary tier.

Player View

Per-player breakdown with tier buttons and completion dates. Completion percentage bar per player.

Auto-detection

Diary completions are detected from the RuneMetrics activity feed on every sync. Any cell can also be toggled manually if the feed entry was missed.

Leaderboards

Boss Kills

Accumulated boss kill counts from the RuneMetrics activity feed. Boss × player matrix with a boss filter dropdown. Auto-updates on every sync.

Firsts

~40 milestone cards showing who achieved each one first in the group — quests, bosses, skill 99s, rare drops. Unachieved cards are dimmed as "Not yet".

Milestones Feed

Full chronological feed of 99s, 120s, quest completions, and notable boss kills.

Skill Mastery

99 and 120 breakdown per player with RS3 wiki skill icons.

Clue Scrolls

Hiscores leaderboard across all 5 tiers (Easy / Medium / Hard / Elite / Master).

Discord Notifications

Connect your group to a Discord channel to receive automatic notifications when things happen in-game. Each group stores its own webhook and event preferences.

Setup

  1. Unlock your group (enter the group secret)
  2. Click the notification bell icon in the header
  3. In Discord: right-click a channel → Edit Channel → Integrations → Webhooks → New Webhook → copy the URL
  4. Paste the URL, choose which events to enable, click Save
  5. Click Test to confirm the connection — a test embed will appear in your Discord channel immediately

Notification Events

EventDefaultTrigger
Level 99s & 120sOnDetected from RuneMetrics activity feed on every sync
Achievement DiariesOnWhen a diary tier completion is detected or manually toggled
Boss First KillsOnFirst time a player's kill count for a boss goes above zero
Group GoalsOnWhen any goal is marked as complete
DropsOffEvery drop auto-detected from activity feed — can be noisy
🔒

The webhook URL is stored server-side and only readable by browsers with the group secret. Only group owners can view or change it.

Self-Hosting

Requirements

  • Node.js 22+
  • Any Linux VPS (Ubuntu 22.04 recommended)
  • Nginx for reverse proxy + SSL
  • PM2 for process management

Local Development

git clone https://github.com/thosjo95/RS3-GIM-Companion.git
cd RS3-GIM-Companion
npm run install:all   # installs root + server + client deps
npm run dev           # API on :3001, Vite dev server on :5173

Open http://localhost:5173 — /api requests are proxied to the backend automatically by Vite.

Seed Gear & Item Data

After first deploy (or after pulling to a new server), seed the RS3 gear and item database so gear slot icons, vault icons, and all item suggestions display correctly:

cd /var/www/RS3-GIM-Companion
node server/scripts/seedRs3Data.js

This seeds 410+ gear items (with requirements and icons) plus 168 skill milestones. It uses INSERT OR REPLACE and is safe to re-run at any time — no restart needed.

Production Deploy

ssh root@YOUR_SERVER_IP
bash /root/deploy.sh
# Script: git pull → npm run build → pm2 restart app

# After deploy, re-seed if gear data changed:
cd /var/www/RS3-GIM-Companion && node server/scripts/seedRs3Data.js
⚡

Backend-only change? Skip the Vite rebuild: git pull && pm2 restart rs3-gim
Use deploy.sh whenever any file under client/ changes. When in doubt, run the full script — it's always safe.

Admin FAQ

Common issues encountered when running the server or managing groups.

Group shows 0 members / members not loading

Symptom: A group was added and claimed, but the member list is empty — even though another unclaimed group with the same name has all the players.

Cause: Players were added to the group before claiming. Claiming created a new database entry with a different id, leaving players attached to the old unclaimed record.

Step 1 — diagnose: Find both groups and note their IDs.

ssh root@YOUR_SERVER_IP
node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const groups = db.prepare(\`
  SELECT id, name, password_hash,
    (SELECT COUNT(*) FROM players WHERE group_id = g.id) AS player_count
  FROM groups g WHERE name LIKE '%YOUR_GROUP_NAME%'
\`).all();
console.log(JSON.stringify(groups, null, 2));
"

You'll see two rows: one with player_count > 0 and no password_hash (OLD), and one with a hash but 0 players (NEW/claimed). Note both id values.

Step 2 — fix: Move players to the claimed group and delete the orphan.

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const moved = db.prepare('UPDATE players SET group_id = NEW_ID WHERE group_id = OLD_ID').run();
const del   = db.prepare('DELETE FROM groups WHERE id = OLD_ID').run();
console.log('Moved:', moved.changes, 'players. Deleted:', del.changes, 'group(s).');
"

Replace OLD_ID and NEW_ID with the values you found. Refresh the app — all members appear immediately.

Activity feed is empty

Cause: The activity feed is populated by a 2-hour background cron (RuneMetrics API). Sync All only updates hiscores — it does not touch RuneMetrics.

Check the last cron run:

pm2 logs rs3-gim --lines 50

To trigger immediately (server-side only):

# Replace 5 with your group's numeric ID
curl -s -X POST http://localhost:3001/api/players/sync-activities/5 | python3 -m json.tool

RuneMetrics returns 502 / 503 errors

Cause: Jagex's RuneMetrics endpoint is temporarily down, or the player's profile is set to Private.

Fix: Ensure each player has their RuneMetrics profile set to Public in-game: Settings → Gameplay → Chat & Social → RuneMetrics Event Log. For server errors, wait and the 2-hour cron will retry automatically.

ℹ️

502 = Jagex server error (just wait). 429 = rate limited (reduce sync frequency). At 1 req/sec with a 2-hour window the app is well within safe limits.

Gear icons missing in the vault or loadouts

Cause: The rs3_items table has not been seeded on this server.

Fix:

cd /var/www/RS3-GIM-Companion
node server/scripts/seedRs3Data.js

The seed script uses INSERT OR REPLACE and is safe to re-run at any time. No restart needed — icons and gear suggestions appear on the next page load.

A group lost their secret / need to re-claim

Fix: Clear the password_hash — this resets the group to unclaimed so they can claim again. All data is untouched.

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
// List groups to find the ID
const groups = db.prepare('SELECT id, name, (SELECT COUNT(*) FROM players WHERE group_id = g.id) AS players FROM groups g ORDER BY id').all();
console.log(JSON.stringify(groups, null, 2));
"

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const result = db.prepare('UPDATE groups SET password_hash = NULL WHERE id = ?').run(GROUP_ID);
console.log('Updated rows:', result.changes);
"

Permanently delete a group

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
const id = GROUP_ID;
db.prepare('DELETE FROM players WHERE group_id = ?').run(id);
const result = db.prepare('DELETE FROM groups WHERE id = ?').run(id);
console.log('Deleted groups:', result.changes);
"

Player shows "Not synced" / hiscores lookup fails

Cause: The RSN was entered with a non-breaking space instead of a regular space — common when copy-pasting from Discord or mobile. Fix by overwriting the RSN with a clean version:

node -e "
const db = require('/var/www/RS3-GIM-Companion/server/database');
db.prepare('UPDATE players SET rsn = ? WHERE id = ?').run('Correct Name Here', PLAYER_ID);
console.log('Done');
"
curl -s -X POST http://localhost:3001/api/players/PLAYER_ID/sync | python3 -m json.tool
